# CURRENT FEATURE DESIGN

The alarm system contains **two sound modes**.

### Mode 1 — Standard Alarm

When this option is selected:

• A normal alarm sound should play
• The sound should loop continuously
• The sound should continue until the user completes the wake-up task

---

### Mode 2 — Speak Title

When this option is selected:

• The alarm should **not play the standard sound**
• Instead the app should **speak the alarm title repeatedly**

Example:

Alarm Title:

"Wake up for morning workout"

When alarm triggers the phone should say:

"Wake up for morning workout"
"Wake up for morning workout"
"Wake up for morning workout"

The title must repeat in a **loop** until the user completes the wake-up task.

---

# CURRENT BUG

Currently the app **always plays the standard alarm sound** even if:

"Speak Title" is selected.

So the alarm mode selection is being ignored.

The alarm trigger system must correctly respect the selected alarm mode.

---

# REQUIRED BEHAVIOR

When the alarm fires, the system must check the stored alarm configuration.

Example:

alarm = {
id,
time,
title,
soundMode
}

Where soundMode can be:

"standard"

or

"speak_title"

---

# ALARM TRIGGER LOGIC

When the alarm triggers:

IF soundMode === "standard"

→ play the normal alarm sound in a loop.

IF soundMode === "speak_title"

→ start text-to-speech loop speaking the alarm title.

---

# STANDARD ALARM IMPLEMENTATION

Use the existing sound playback system.

Requirements:

• sound must loop
• sound must be loud and continuous
• must stop only when task is completed

Ensure sound stops safely when:

• alarm task is completed
• screen unmounts
• user stops alarm

---

# SPEAK TITLE IMPLEMENTATION

Use:

expo-speech

Implementation requirements:

• speak the alarm title
• when speech finishes → repeat automatically
• continue looping until alarm task is completed

Example safe pattern:

Speech.speak(title, {
onDone: repeatSpeech,
onStopped: stopLoop
})

Where repeatSpeech triggers the next speak call.

---

# IMPORTANT SAFETY RULES

Prevent these problems:

• multiple speech loops running simultaneously
• infinite recursion causing crash
• memory leaks
• speech continuing after alarm ends

When alarm stops:

Speech.stop()

must be called.
