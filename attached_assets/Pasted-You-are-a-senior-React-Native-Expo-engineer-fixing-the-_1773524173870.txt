# PART 2 — Discipline Count Logic

Discipline Count should increase when the user **successfully completes the wake-up task**.

Example:

Before task completion:

disciplineCount = 5

After successful alarm completion:

disciplineCount = 6

Rules:

• Increase by **+1 only once per alarm completion**
• Prevent duplicate increments
• Persist updated value in AsyncStorage

---

# PART 3 — Streak Count Logic

Streak represents **consecutive days of successful wake-ups**.

Rules:

If the user completes today's alarm:

→ compare today's date with lastCompletionDate.

Cases:

1️⃣ If lastCompletionDate was **yesterday**

→ streakCount += 1

2️⃣ If lastCompletionDate was **today**

→ do NOT increase streak again.

3️⃣ If lastCompletionDate is **older than yesterday**

→ streak is broken.

If streak breaks:

→ reset streakCount to **0**

Then if user completes a new alarm:

→ start streak again at **1**

---

# PART 4 — Streak Freeze / Snap Feature

UNSNWOOZE includes a **streak protection feature**.

If the user achieves **3 consecutive days of streak**, they earn **1 Streak Freeze (Snap)**.

Example:

Day 1 → streak = 1
Day 2 → streak = 2
Day 3 → streak = 3

User earns:

streakFreezeCount += 1
 like first streak freeze at 3 another at 9 so on , they can collect and freeze their streak for 24 hour

---

# PART 5 — Using a Streak Freeze

If the user **misses a day**, normally the streak resets.

But if the user has a **streakFreeze available**:

Instead of resetting streak:

• consume one freeze
• streak remains unchanged

Example:

streak = 5
freeze = 1

User misses a day.

Result:

streak stays 5
freeze becomes 0

If no freeze is available:

→ streak resets to 0.
