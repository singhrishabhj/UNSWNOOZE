Build a modern mobile-first web application called **UNSNWOOZE – Smart Alarm & Anti-Snooze Productivity App**.

The purpose of this application is to prevent users from repeatedly snoozing alarms and help them build disciplined morning routines.

The product must feel like a premium productivity application with a clean, minimal interface. The design should be calm, modern, and visually refined while remaining extremely simple to use.

Prioritize **design quality, smooth UI interactions, and minimalistic layout**.

---

TECH STACK

Frontend: React
Styling: TailwindCSS
Backend: Node.js + Express
Database: Supabase or Firebase
Camera Access: Browser Camera API
Voice: Web Speech API

Make the application installable as a **Progressive Web App (PWA)** so it behaves like a mobile application.

---

The interface should follow these principles:

Minimal interface

The application should feel calm, focused, and premium.

---

ALARM SOUND TYPE

Allow users to select how the alarm behaves.

Option 1
Standard Alarm Sound

A repeating alarm tone plays until the wake-up task is completed.

Option 2
Speak Alarm Title

Instead of a ringtone, the application repeatedly speaks the alarm title using text-to-speech.

Example:

Alarm Time: 6:00 AM
Alarm Title: Morning Workout

If Speak Alarm Title is enabled, the system will repeatedly say:

"Morning Workout. Wake up."

Speech should repeat every 4–6 seconds until the user completes the wake-up task.

Use the Web Speech API for speech synthesis.

WAKE-UP TASKS

For MVP simulate verification logic such as:

---

TOOTHPASTE PHOTO VERIFICATION

Open camera.

User must capture a photo of toothpaste.

The alarm stops only after the photo is captured.

This forces the user to physically get out of bed.
